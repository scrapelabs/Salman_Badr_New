Automation

          
Schedule with GitHub Actions

        

        Webhook
      


      

        MatchMiner doesn't run scrapes on a timer itself — it exposes a secure
        trigger endpoint that any scheduler can call. The steps below wire up a
        free GitHub Actions cron job that launches {{ s.code }} on
        whatever cadence you choose. Each call starts exactly one run — the same as pressing
        “Start” on the Real-time tab — and respects maintenance mode and the one-run-at-a-time rule.
      


      

        

          
          

            
            
          

          
A POST here launches a run. Authenticate with the token below.

        

        

          
          

            
            
            
          

          
Treat this like a password — anyone with it can start a run. Rotate it below if it leaks.

        

      


      
    
        
  1. 
          
    Add the token as a repository secret
    
          
    
            In your GitHub repo open Settings → Secrets and variables → Actions →
            New repository secret. Name it {{ secret_name }} and paste the
            trigger token above as the value.
          
    
        
    2. 
        
  2. 
          
    Add the workflow file
    
          
    
            Create .github/workflows/{{ workflow_filename }} in your repo with the
            contents below, then commit it to your default branch.
          
    
          
    
            
            
    {{ workflow_yaml }}
    
          
    
        
    3. 
        
  3. 
          
    Pick your cadence
    
          
    
            The cron line controls when it runs (UTC). "0 6 * * *" is daily at
            06:00. Other examples: "0 * * * *" hourly, "0 6 * * 1" Mondays at
            06:00, "0 6,18 * * *" twice a day. Build one at
            crontab.guru.
          
    
        
    4. 
        
  4. 
          
    Run it on demand to test
    
          
    
            Open the Actions tab in your repo, choose
            MatchMiner — {{ s.code }} scheduled scrape, and click Run workflow
            (you can set the year). A new run appears under
            Calls history within a minute.
          
    
        
    5.

curl -fsS -X POST "{{ trigger_url }}" \

  -H "Content-Type: application/json" \
